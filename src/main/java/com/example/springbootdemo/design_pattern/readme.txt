# 设计模式7大原则
1.单一职责原则
2.接口隔离原则
3.依赖倒转（倒置）原则
4.里氏替换原则(继承时最好不要重写父类方法)
5.开闭原则(拓展开放,修改关闭) OCP原则
6.迪米特法则(最小知道原则:降低类耦合)
7.合成复用原则(尽量使用合成/聚合的方式,而不是继承)
总结:1.找出应用中可能变化之处,把它们独立出来,不应该和不需要变换的代码混在一起
     2.面向接口编程,而不是面对实现编程
     3.为了交互对象之间的松耦合努力
